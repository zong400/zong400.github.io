calico tunl ip冲突

By Zong

2019-12-31

Contents

集群新增2个node，完了之后发现网络有问题，其中一个节点和pod网络不通，经过一轮排查，发现新增的两个node分配的tunl0 ip是一样的，就是出现问题的原因。
如图：

通过 calicoctl get node 和 kubectl get no 对比发现节点域名不一致

解决方法：
删除 /var/lib/calico/nodename，重启对应节点的calico node
此问题疑似由于node8是克隆自node1，待验证
