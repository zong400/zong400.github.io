Kubernetes 1.8 集群网络

By Zong

2017-12-13

Contents

Kubernetes的网络模型假设所有pods都处在一个可以互通的二层网络，这在GCE、AWS等云端是天然实现的。但是我们自己搭建的集群就不可能了，所以需要通过第三方的网络组件实现不同物理机上的pods网络互通。
基本上开源的网络组件有Flannel、waves、Open vSwitch和Calico，Flannel和waves的实现方式是模拟一个大二层网络（Overlay Network），通过网桥把数据包封装、解封的方式发送到目标pods。Calico的实现方式是三层路由方案，它在每个物理机实现一个vRouter转发数据包，通过网桥把数据包封装、解封的方式发送到目标pods。

本次选择Calico搭建网络，安装需满足以下条件：

  • kubelet需要配置使用CNI网络插件 (e.g –network-plugin=cni)
  • kube-apiserver、kubelet需要配置特权模式参数 –allow_privileged=true
  • kube-proxy 不能使用–masquerade-all 参数
  • kubernetes版本个人推荐1.6以上

安装calico在kubernetes 1.6以上并不复杂，官方有yaml配置，下载下来改一下参数就能直接使用kubectl生成容器。calico node会以DaemonSet 形式部署在每一台kube node上。
下载calico.yaml配置模板并修改内容。

  1. 分配给Pod的IP段
    默认ip段如果与当前已有ip冲突，可以修改 CALICO_IPV4POOL_CIDR 部分

  2. 配置IP-in-IP
    IP-in-IP默认开启，采用teunl方式跨子网，以下几种情况可以关闭IPIP（CALICO_IPV4POOL_IPIP 设置OFF）

    运行在aws
    所有kubernetes node运行在同一个L2网络
    使用BGP方式让Pod IP在底层通信

  3. 设置etcd
    etcd作用是存储ip和policy策略，默认不使用ssl通信，没有认证。
    修改参数etcd_endpoints为正确的etcd地址。

  4. 设置kubeconfig

  5. k8s_api_root
    填写ClusterIP的第一个地址，使用sudo kubectl get svc查看

完成后使用 kubectl apply -f calico.yaml 命令就能安装好calico

完整的calico.yaml

使用指定版本的calico/node，calico/cni等容器可以避免官方更新引起的前后版本不一致问题

在腾讯云等使用VPC网络的环境，需要用IPIP模式
